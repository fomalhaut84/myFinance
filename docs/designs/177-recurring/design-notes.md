# 반복 거래 디자인 노트 — #177

## 디자인 방향

기존 가계부 UI 패턴 재사용. 반복 규칙 관리에 집중한 단순한 CRUD 페이지.

## 페이지 구조

### /recurring 페이지
- 헤더: "반복 거래" + "자동 생성되는 정기 내역"
- "+ 반복 거래 추가" 버튼 (sodam)
- 반복 거래 목록 테이블

### 테이블 컬럼
- 활성: 초록/회색 원 토글 (클릭으로 on/off)
- 카테고리: 아이콘 + 이름
- 내용: 설명
- 금액: 빨간색
- 주기: "매월 N일", "매주 X요일", "매년 N월"
- 다음 실행: nextRunAt 날짜
- 액션: 수정/삭제

### 비활성 행
- opacity 0.5로 시각적 구분
- 정렬: nextRunAt 오름차순

## 폼 (슬라이드 패널)

### 주기 선택
- 세그먼트 컨트롤: 매월 | 매주 | 매년
- 주기에 따라 실행일 입력 변경:
  - 매월: 날짜 select (1~31 + 말일)
  - 매주: 요일 버튼 그리드 (월~일, 단일 선택)
  - 매년: 월 select + 날짜 select

### 시작일
- nextRunAt 초기값
- date input

## 삭제 모달
- 카테고리, 내용, 금액, 주기 표시
- "삭제하면 이후 자동 생성이 중단됩니다. 이미 생성된 내역은 유지됩니다."
