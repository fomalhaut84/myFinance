# 커스텀 전략 웹 UI 디자인 노트

## 목적
봇 커맨드에만 존재하는 커스텀 전략 관리를 웹에서 시각화. 자연어 등록 → 파싱 미리보기 → 확정 → 카드 그리드로 상태/이력 관찰.

## 컬러 매핑
- 활성 상태: `--sejin` (#34d399 / #059669) — 다른 페이지 계좌 소유자 컬러와 겹치지만, 상태 표시로만 사용
- 빈도 chip:
  - `once` — `--amber` (긴급/일회성 강조)
  - `daily` — `--sodam` (정기)
  - `always` — `--dasom` (지속)
- 논리 `AND`/`OR` chip — `--sodam`

## 레이아웃
1. **Sticky Header**: 페이지 타이틀 + 알림 on/off 배지 + 테마 토글
2. **등록 카드** (상단): 
   - 자연어 textarea (500자, 실시간 char count)
   - "미리보기" 버튼 → 파싱 결과 인라인 표시 → "등록" 확정
   - 예시 3개 (클릭 시 자동 채움)
3. **필터 셀렉트**: 전체 / 활성만 / 비활성만
4. **카드 그리드**: 1열 (모바일) / 2열 (태블릿) / 3열 (데스크톱)
5. **알림 off 배너** (조건부): AlertConfig.custom_strategy_alerts=off 일 때만 표시
6. **면책 문구** (하단)

## 카드 구조
- 헤더 좌: 상태 dot + 이름 + ticker + 등록일
- 헤더 우: 활성 토글 스위치
- 조건 목록: 모노스페이스 폰트, AND/OR 논리 인라인
- 푸터: 빈도 chip + 최근 발동 chip + 편집/삭제 버튼
- 비활성 카드: opacity 0.55, status dot 회색

## 인터랙션
- **등록 흐름**: 텍스트 입력 → 미리보기 → 파싱 결과 검증 → 등록 or 취소
  - 파싱 실패 시: 에러 인라인 표시 + 재조정 안내
  - 파싱 지연 (~30-60s): 스피너 + "AI 파싱 중" 표시
- **활성 토글**: 즉시 PUT (낙관적 UI)
- **편집 모달**: 이름 / 빈도 / 논리 / 활성 상태 편집 (조건 자체는 불가)
- **삭제**: 기존 `DeleteModal` 재활용 (프로토타입엔 미포함)

## 반응형
- 모바일 (<768px): 카드 1열, 폼 여백 축소
- 태블릿 (>=768px): 카드 2열
- 데스크톱 (>=1200px): 카드 3열, 최대 폭 1200px

## 접근성 & 사용성
- 상태 dot + 색상 + 텍스트 배지 (색맹 대비 이중 표시)
- 액션 버튼은 최소 32px hit area
- 자연어 미리보기 결과를 눈에 잘 띄게 (배경 강조)
- 조건 자체 편집 불가 안내를 편집 모달에서 명시

## 폼 검증
- 활성 상한 50개 도달 시: chip 색상 변경 + 등록 버튼 비활성
- textarea 500자 초과: char count 빨간색

## 디자인 상 결정 사항
- 카드에 발동 이력 카운트 표시하지 않음 (v1 은 마지막 시각만) — 로그 페이지는 별도 이슈
- 조건 그래픽 빌더 X — 자연어 파싱만 노출 (파싱 실패 시 표현 조정 유도)
- 웹 알림 (브라우저 push) X — 텔레그램만
