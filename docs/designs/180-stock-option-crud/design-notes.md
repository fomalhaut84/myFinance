# 스톡옵션 CRUD 디자인 노트 — #180

## 디자인 방향

기존 StockOptionDashboard에 추가/수정/삭제 기능 확장. RSU CRUD와 동일 패턴(슬라이드 폼 + 삭제 모달).

## 변경 영역

### StockOptionDashboard 확장
- 헤더에 "+ 스톡옵션 추가" 버튼 (sodam)
- 각 옵션 카드에 수정(연필) / 삭제(휴지통) 아이콘

### StockOptionForm (슬라이드 패널)
- 필드: 계좌, 종목(ticker/displayName), 부여일, 만료일, 행사가격, 총부여수량, 메모
- 수정 모드: 기존값 prefill

### StockOptionDeleteModal
- 옵션 정보 표시 (종목, 행사가, 부여수량)
- "삭제 시 행사 스케줄도 함께 삭제됩니다."

### Vesting 관리 (인라인)
- 각 옵션 카드 내 행사 스케줄 목록
- "+ 스케줄 추가" 버튼
- 행별 수정/삭제 (exercised 상태는 비활성)
- VestingForm: 소형 모달 (행사가능일, 수량)

## UI 컴포넌트

| 요소 | 구현 방식 |
|------|----------|
| 옵션 추가/수정 | 우측 슬라이드 패널 420px |
| 옵션 삭제 | 중앙 확인 모달 |
| 스케줄 추가/수정 | 인라인 폼 또는 소형 모달 |
| 스케줄 삭제 | 즉시 삭제 (확인 불필요, 간단) |
