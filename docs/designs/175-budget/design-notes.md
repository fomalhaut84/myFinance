# 예산 관리 디자인 노트 — #175

## 디자인 방향

기존 디자인 시스템 유지. 가계부 페이지와 일관된 레이아웃.
예산 진행률을 시각적으로 명확하게 전달하는 것이 핵심.

## 페이지 구조

### 헤더 영역
- 타이틀: "예산" + 서브텍스트
- 월 네비게이션: < 2026년 3월 > (이전/다음 화살표)
- "예산 복사 →" 버튼 (sodam 컬러)

### 전체 예산 요약 카드
- 대형 금액 표시 (28px, --bright)
- 소비액 (--red) + 잔여액 (--emerald)
- 프로그레스 바: 단색, 비율에 따라 색상 변경
- "금액 수정" 버튼 (우측 상단)

### 카테고리별 예산 테이블
- 그리드 레이아웃: 카테고리명 | 진행률 바+정보 | 소비액 | 잔여액 | 액션
- 진행률 색상 규칙:
  - <70%: 초록 (--emerald)
  - 70~100%: 노랑 (--yellow)
  - >100%: 빨강 (--red) + "초과" 텍스트
- 미설정 카테고리: 회색 + "설정" 버튼
- 하단: "+ 카테고리 예산 추가" 링크 버튼

## 인터랙션

### 인라인 편집
- 연필 아이콘 클릭 → 진행률 영역이 금액 입력 필드로 교체
- 저장/취소 소형 버튼
- POST /api/budgets (upsert)

### 예산 복사
- "예산 복사" 클릭 → 확인 모달
- 현재 월의 전체 + 카테고리 예산 목록 표시
- "복사" 확인 → 다음 달에 동일 예산 upsert

## UI 컴포넌트

| 요소 | 구현 방식 |
|------|----------|
| 월 네비게이션 | 좌/우 화살표 버튼 + 중앙 라벨 |
| 전체 예산 카드 | 단일 카드, 프로그레스 바 |
| 카테고리 예산 | 그리드 테이블 (데스크톱), 카드 스택 (모바일) |
| 예산 편집 | 인라인 입력 (행 내 교체) |
| 예산 복사 | 중앙 확인 모달 |
| 예산 추가 | 카테고리 선택 드롭다운 + 금액 입력 |

## 색상 참조

- 진행률 <70%: `var(--emerald)` #34d399
- 진행률 70~100%: `var(--yellow)` #fbbf24
- 진행률 >100%: `var(--red)` #f87171
- 소비액: `var(--red)` #f87171
- 잔여액 양수: `var(--emerald)` #34d399
- 잔여액 음수: `var(--red)` #f87171
- 예산 복사/추가 버튼: sodam 컬러
