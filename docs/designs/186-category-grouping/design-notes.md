# 카테고리 그루핑 디자인 노트 — #186

## 디자인 방향

기존 카테고리 관리 UI와 예산 UI를 확장. 그룹이라는 상위 계층을 추가하되, 기존 UX를 최소한으로 변경.

## 변경 영역

### 1. /categories 페이지
- 카테고리 테이블: 그룹별로 시각적 그룹핑 (그룹 헤더 행 + 들여쓰기)
- 그룹 헤더: 접기/펼치기, 아이콘+이름+카테고리수 뱃지
- 수입 탭: 그룹 없이 기존 플랫 리스트 유지
- 그룹 관리 섹션: 카드 그리드 (아이콘+이름+카테고리수+수정/삭제)
- 그룹 추가: 인라인 폼 (이름, 아이콘, 정렬 순서)

### 2. 카테고리 편집 패널
- 기존 필드 유지 + "그룹" select 드롭다운 추가
- 옵션: 미분류 + 10개 그룹 (아이콘 포함)
- 수입 카테고리: 그룹 필드 숨김

### 3. /budgets 페이지
- 기존 "카테고리별 예산" → "그룹별 예산"으로 확장
- 그룹 행: 접기/펼치기, 그룹 전체 예산 + 진행률
- 펼치면 하위 카테고리별 소비 분배 표시 (읽기 전용)
- 그룹 예산: Budget.groupId로 독립 관리

### 4. /expenses 페이지
- CategoryPieChart에 "카테고리별 | 그룹별" 토글 추가
- 그룹별: 카테고리를 그룹으로 합산, 그룹 아이콘+이름으로 범례

## UI 컴포넌트

| 요소 | 구현 방식 |
|------|----------|
| 그룹 헤더 행 | 다른 배경색 + 아이콘 + 접기/펼치기 |
| 그룹 관리 카드 | 카드 그리드 (auto-fill, 160px min) |
| 그룹 추가/수정 | 인라인 폼 (이름+아이콘+정렬) |
| 그룹 select | CategoryEditPanel 내 dropdown |
| 그룹별 예산 | 접기/펼치기 행 + 하위 카테고리 표시 |
| 그룹별 파이차트 | 기존 파이차트 + 토글 전환 |

## 삭제 규칙
- 카테고리가 있는 그룹: 삭제 불가 (비활성, opacity 0.3)
- 빈 그룹만 삭제 가능
