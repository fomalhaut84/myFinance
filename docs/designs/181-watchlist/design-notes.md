# 관심종목 웹 관리 디자인 노트 — #181

## 디자인 방향

텔레그램에서만 관리하던 관심종목을 웹에서도 관리. /watchlist 신규 페이지.
기존 CRUD 패턴 (테이블 + 슬라이드 폼 + 삭제 모달) 재사용.

## 페이지 구조

### /watchlist 페이지
- 헤더: "관심종목" + "N종목"
- "+ 관심종목 추가" 버튼 (sodam)
- 테이블: 종목명 | 시장 | 전략 | 현재가 | 목표 매수가 | 매수 구간 | 메모 | 액션

### WatchlistTable
- PriceCache 활용 현재가 표시
- 전략 뱃지 (swing=파랑, momentum=주황, value=초록, scalp=빨강)
- 매수 구간 표시 (low ~ high)
- 현재가 vs 목표가 비교 (목표가 이하면 초록 강조)
- 수정/삭제 아이콘

### WatchlistForm (슬라이드 패널)
- 필드: 종목티커, 종목명, 시장(US/KR), 전략(select), 목표 매수가, 매수 구간(하한/상한), 메모
- 수정 모드: prefill

### 삭제
- 간단 확인 (종목명 표시 + 확인)

## nav-config
- 포트폴리오 그룹에 "관심종목" 추가 (👀)
