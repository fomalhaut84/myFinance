# Phase 22-A: AI 시스템 안내 강화

## 목적

AI 어드바이저가 자체 시스템 기능을 정확히 안내하지 못하는 문제 해결.
또한 향후 Phase 23-24에서 추가될 쓰기 도구의 사용 규칙을 미리 정의.

## 문제 사례

사용자: "관심종목 매수구간 도달 알람 설정 못하나?"
AI: "❌ 자동 알람: 2차 마일스톤 대기 중 / ❌ watchlist 구간 알람: 현재 지원 안 함"

→ **실제로는 이미 구현됨** (PR #234, 봇 커맨드 `관심 [종목] 매수구간 [하한] [상한]`).
원인: AI가 옛 CLAUDE.md를 직접 참조 + 봇 기능을 시스템 프롬프트에서 안내받지 못함.

## 요구사항

- [ ] 시스템 프롬프트에 텔레그램 봇 커맨드 목록 추가
- [ ] 알림 시스템 현황 명시 (목표가/매수구간/TA 시그널/거래시간 필터링 등)
- [ ] CLAUDE.md/docs 직접 참조 금지 — 시스템 프롬프트와 MCP 도구만 신뢰
- [ ] 쓰기 도구 사용 규칙 추가 (Phase 23-24 대비):
  - 추가/수정 → "다음과 같이 등록할까요?" 확인 후 실행
  - 삭제 → 사용자 명시적 동의 필요
  - 조회는 즉시 실행

## 변경 파일 (1개)

`src/lib/ai/system-prompt.ts` — SYSTEM_PROMPT 확장

### 추가할 섹션

1. **텔레그램 봇 커맨드** (관심종목/전략/매매 등)
2. **알림 시스템 현황** (현재 동작 중인 모든 알림 종류)
3. **시스템 정보 안내 규칙** (CLAUDE.md 비참조)
4. **쓰기 도구 사용 규칙** (확인 절차)

## 테스트 계획

- [ ] AI에게 "매수구간 알림 설정 가능?" → 봇 커맨드 안내
- [ ] AI에게 "TA 시그널 알림은?" → 자동 전송 중임을 안내
- [ ] AI가 임의로 데이터 수정 시도 → 사용자 확인 절차 따름

## 제외 사항

- 실제 쓰기 도구 구현 (Phase 23-24)
- 봇 커맨드 신규 추가 없음 (안내만)
