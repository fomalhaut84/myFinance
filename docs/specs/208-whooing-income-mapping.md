# 수입 카테고리 후잉 웹훅 매핑 누락 수정

## 목적

수입 거래가 후잉에 전송되지 않는 버그 수정.
원인: 후잉 설정 UI(`WhooingSettings.tsx`)에서 카테고리 매핑 목록을 expense 타입만 필터링하고 있어,
수입 카테고리에 대한 매핑을 설정할 수 없음.
매핑이 없으면 `left`가 빈 문자열로 전송되어 후잉에서 정상 처리되지 않음.

## 요구사항

- [ ] 후잉 설정 카테고리 매핑 테이블에 수입 카테고리도 표시
- [ ] 소비/수입 카테고리를 그룹 구분하여 표시 (가독성)
- [ ] 기존 소비 카테고리 매핑 동작 회귀 없음

## 기술 설계

### 변경 파일 (1개)

**`src/components/settings/WhooingSettings.tsx`**
- L57: `c.type === 'expense'` 필터 제거 → 전체 카테고리 로드
- 매핑 테이블: 소비/수입 그룹 구분 (소제목 행 또는 optgroup 스타일)

### API 변경: 없음

`WhooingCategoryMap`은 `categoryId`로만 매핑하며 타입 제한이 없음.
매핑 저장/조회 API도 타입 무관하게 동작.

## 테스트 계획

- [ ] 후잉 설정 → 카테고리 매핑 테이블에 소비/수입 모두 표시
- [ ] 수입 카테고리에 매핑 설정 후 저장 → 정상 저장 확인
- [ ] 수입 거래 입력 → 후잉 웹훅 전송 확인
- [ ] 기존 소비 매핑 정상 동작 (회귀)
- [ ] lint + typecheck + build 통과
