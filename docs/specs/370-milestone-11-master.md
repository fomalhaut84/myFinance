# 11차 마일스톤 — 능동 AI + 알림/매매 정밀화

- **작성일**: 2026-07-01
- **타입**: 마일스톤 (마스터 스펙, 후속 sub-phase 로 분할 진행)
- **참조**: 이슈 #230 (4개월 미해결 사용자 요구), `memory/project_next_milestone_11.md`

## 1. 배경

10차 마일스톤까지 안정성 (envelope + 봇 안정성) 을 마쳤고, v0.8.2 로 RSU vest 자동화 + 베스팅 캘린더 개선 완료. 이제 사용자 실사용 관점의 **UX/알림/AI 능동성** 확장.

핵심 힌트: **이슈 #230** (2026-04-02 미해결) 이 명시적으로 다음을 요구:
1. 관심종목 시세 표시 (현재 "시세 없음" 만 나옴)
2. 투자 목적별 알람 (전략 태그별 다른 트리거)
3. 특정 종목 목표가 알림 (예: SOXL $40)
4. AI 매매 가이드 (전략별 실시간 지침)
5. 스킬 업로드형 전략 실행

**정리한 결과**:
- (3) 목표가 알림은 이미 구현됨 (`holdingStrategy.targetPrice`, `watchlist.targetBuy`) — UX 노출 개선 필요
- (1) 시세 버그는 즉시 진단/수정 대상
- (2), (4), (5) 는 신규 개발 영역

## 2. 목표

이번 마일스톤 종료 시:
- ✅ 관심종목이 봇에서 시세와 함께 표시됨
- ✅ 전략 태그별로 스마트 알림 트리거 (스윙/모멘텀/장기 각각 다른 조건)
- ✅ TA 시그널 발생 시 AI 가 종목별 매매 판단 조언 첨부
- ✅ 아침/저녁 cron 이 AI 로 포트폴리오 능동 리뷰 → 텔레그램 알림
- ✅ 사용자가 커스텀 전략을 텍스트로 정의하고 AI 가 그 기준으로 감시

## 3. Sub-Phase 분할

### Phase 29-A: 관심종목 시세 버그 fix + 진단 (작음, 즉시)

**목적**: 이슈 #230 (1) 해결. 봇 `관심` 명령 + 웹 `/watchlist` 페이지에서 시세 정상 표시.

- [ ] 원인 진단: price-fetcher 는 관심종목 티커 포함하는데도 왜 "시세 없음" 나오는지 (로그 추가 + 재현)
- [ ] fix: 티커 정규화 / 시장 미확정 / yahoo 조회 실패 케이스 처리
- [ ] fetchQuote fallback 로그 강화 (sanitizeError 경유)

**노력**: XS. 며칠 내 마무리.

### Phase 29-B: 전략별 스마트 알림

**목적**: 이슈 #230 (2). 관심종목/보유종목의 `strategy` 필드 (long_hold/swing/momentum/value/watch/scalp) 별로 다른 알림 규칙.

전략별 트리거 예시:
| 전략 | 트리거 |
|---|---|
| long_hold | 급락 -5% (매수 기회) |
| swing | RSI < 30 (과매도) or > 70 (과매수) — 이미 TA 시그널 있음 |
| momentum | 20일 이평선 돌파 상승 |
| value | PE 낮은 상태 유지 + 급락 (감시자 관심) |
| watch | 어떤 시세 변화든 (전략 미정, 관찰만) |
| scalp | 당일 지지/저항 근접 |

- [ ] `HoldingStrategy.strategy` / `Watchlist.strategy` 별 트리거 규칙 정의 (설정 UI 최소)
- [ ] cron 스케줄러 확장 (기존 price-alert 재활용)
- [ ] 알림 메시지 포맷: 전략 + 트리거 사유 + 다음 행동 힌트

### Phase 29-C: AI 매매 가이드 실시간

**목적**: 이슈 #230 (4). TA 시그널 발생 → AI 가 stock-trading-method 스킬 기반 종목별 판단 + 진입/청산 힌트 첨부.

- [ ] TA 시그널 알림 (`ta-signal-alert.ts`) 발송 시, 해당 종목별 AI 짧은 가이드 요청 (`askAdvisor`) 후 메시지에 첨부
- [ ] AI 프롬프트: 종목 정보 + TA 데이터 + 사용자 전략 → 3~5줄 조언
- [ ] 예산: TA 시그널 하루 몇 건 발생하는지에 따라 max-budget 설정
- [ ] fallback: AI 실패 시 TA 시그널만 발송 (기존 흐름)

### Phase 29-D: 능동 AI 인사이트 cron

**목적**: 현재는 사용자 질문 시만 응답. 능동적으로 하루 시장/포트폴리오 관찰 → 알림.

- [ ] 아침 브리핑 강화 (기존 브리핑에 AI 리뷰 추가 vs 별도 cron 신규)
- [ ] 저녁 클로징 리뷰 (KRX/미국 장 마감 후 요약 + 내일 관찰 포인트)
- [ ] AI 프롬프트에 능동 관찰 지침 (사용자 질문 없이 어드바이저 관점)
- [ ] 사용자 on/off (`alertConfig` 에 `active_ai_review: true/false`)

### Phase 29-E: 커스텀 전략 스킬 업로드

**목적**: 이슈 #230 (5). 사용자가 텍스트로 매매 전략 정의 → AI 가 그 기준으로 감시.

- [ ] `CustomStrategy` 신규 모델 (name, description, tickers, criteria, notifyFrequency)
- [ ] 웹 UI: 전략 등록 폼 (자연어 or 구조화)
- [ ] cron: 등록된 전략마다 AI 호출 → 조건 만족 시 알림
- [ ] MCP 도구 확장: AI 어드바이저가 사용자 대화 중 전략 등록 지원

## 4. 진행 순서 (추천)

1. **29-A** 먼저 (버그 fix, 짧음, 즉시 가치)
2. **29-B** — 알림 시스템 확장 (기존 price-alert 기반)
3. **29-C** — AI + TA 통합 (기존 인프라 활용)
4. **29-D** — 능동 cron (인프라 안정 확인 후)
5. **29-E** — 커스텀 전략 (별도 phase, 데이터 모델 추가)

## 5. 성공 지표

- 관심종목 화면에 실시간 시세 100% 표시

- TA 시그널 알림에 AI 조언 첨부율 > 80%
- 아침/저녁 능동 리뷰가 실제 유용한 인사이트 포함 (사용자 피드백 기반)
