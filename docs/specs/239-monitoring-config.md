# 전략 종목 주기적 모니터링

## 목적

TA 시그널 모니터링 주기를 텔레그램에서 설정 가능하게 하고,
시그널 발생 시 AI가 매매 전략 스킬 기반 간략 가이드를 포함하여 알림.

## 요구사항

- [ ] 텔레그램 `/알림설정 모니터링주기 [분]` 커맨드로 TA 체크 주기 변경
- [ ] AlertConfig DB에 `ta_check_interval_min` 키로 저장 (기본 10분 = 시세 갱신 주기)
- [ ] 시그널 발생 시 AI 간략 가이드 포함 (askAdvisor 호출, 스킬 참조)
- [ ] AI 가이드 실패 시 기존 시그널 알림만 발송 (fallback)

## 기술 설계

### 변경 파일

1. **`src/bot/commands/alert.ts`** — 모니터링주기 설정 커맨드 추가
2. **`src/bot/notifications/ta-signal-alert.ts`** — 시그널 발생 시 AI 가이드 추가
3. **`src/lib/cron.ts`** — TA 체크 주기를 DB에서 동적 조회

### 모니터링 주기 동작

현재 시세 갱신은 10분 고정. TA 체크는 시세 갱신 후 실행.
→ TA 체크 주기 = 시세 갱신 주기와 동일 (10분).
사용자가 30분으로 설정하면, 시세 갱신 3번 중 1번만 TA 체크 실행.

### AI 가이드 포함

시그널이 발생한 종목에 대해 AI에게 간략 분석 요청:
- "SOXL에서 RSI 과매도 시그널 발생. stock-trading-method 프레임워크로 현재 상황 1~2줄 가이드."
- 타임아웃 30초, 실패 시 시그널만 발송
