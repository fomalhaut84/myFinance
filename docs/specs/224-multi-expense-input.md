# 텔레그램 복수 가계부 입력

## 목적

한 메시지로 여러 가계부 항목을 입력할 수 있도록 지원.
예: "3월 29일 외식 20만원, 오늘 커피 800원, 어제 영화 15만원"
→ 3건의 거래가 각 날짜에 맞게 생성됨.

## 요구사항

- [ ] 복수 거래 자연어 파싱 (AI 활용)
- [ ] 각 건에 날짜, 금액, 내용, 타입(소비/수입) 추출
- [ ] 상대 날짜 지원 (오늘, 어제, 그저께, N일 전, 특정 날짜)
- [ ] 각 건에 카테고리 자동 매칭
- [ ] 일괄 확인 키보드 → 일괄 저장 + 후잉 전송
- [ ] 기존 단일 건 입력 로직 유지 (회귀 없음)

## 기술 설계

### 트리거 조건

기존 `parseExpenseInput`이 "금액이 모호합니다" 에러 반환 시 → 복수 건 AI 파싱으로 전환.
즉, 숫자 토큰이 2개 이상인 메시지가 복수 건 후보.

### AI 파싱 프롬프트

askAdvisor에 복수 거래 파싱 프롬프트 전송. JSON 배열 반환:
```json
[
  {"date": "2026-03-29", "description": "외식", "amount": 200000, "type": "expense"},
  {"date": "2026-03-31", "description": "커피", "amount": 800, "type": "expense"},
  {"date": "2026-03-30", "description": "영화", "amount": 150000, "type": "expense"}
]
```

### 확인 + 저장

파싱 결과를 사용자에게 보여주고 확인/취소 키보드 표시.
확인 시 각 건에 카테고리 자동 매칭(matchCategory) → 일괄 저장 + 후잉 전송.

### 변경 파일

1. **`src/bot/commands/expense.ts`** — 복수 건 AI 파싱 함수 + 확인/저장 로직
2. **`src/lib/ai/claude-advisor.ts`** (변경 없음, 기존 askAdvisor 사용)
